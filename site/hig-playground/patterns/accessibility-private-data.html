✓ 이렇게 하세요

  • 실제로 필요한 시점에 권한 요청
  • 명확한 사용 목적 설명 제공
  • 거부 시 대안 기능 제공
  • Pre-prompt으로 사용자 준비

✗ 이렇게 하지 마세요

  • 앱 시작 시 모든 권한 한꺼번에 요청
  • 모호한 사용 목적 설명
  • 권한 거부 시 앱 사용 차단
  • 시스템 다이얼로그만 의존

iPhone 미리보기에서 나쁜 패턴을 직접 확인해보세요

iOS에서 개인 데이터에 접근하려면 반드시 사용자 동의를 받아야 합니다. 사전 안내 화면(Pre-prompt)으로 맥락을 제공하고, 명확한 사용 목적 설명(Usage Description)을 작성하며, 권한 거부 시에도 대안 기능(Graceful Degradation)을 제공하는 것이 좋습니다.
